## 1. Brand essence

> «Устойчивая основа для движения вперёд.»
> «Снижаем цену прогресса, создавая фундамент для инноваций наших клиентов.»

**Характер бренда (4 атрибута):**
- Сильный и динамичный
- Профессиональный и уверенный
- Современный и прогрессивный
- Проактивный и харизматичный

**Что это значит для слайдов:**
- Один смысл на слайд (минимум воды)
- Цифры подаются крупно («чем крупнее цифры — тем эффектнее подана информация» — стр.33)
- Без декоративных украшательств — паттерны Cloud.ru сами по себе декор
- Тёмные акценты только для эмфазы, общий тон **светлый**

## 2. Look & Feel — стилистические инварианты

Из брендбука стр.6 «Look & feel»:

1. **Прямые углы** — везде где возможно
2. **Окончание штриха квадратное** (не round caps)
3. **Без скруглений** — только если это часть метафоры/объекта
4. **Гротеск, а НЕ гуманизм** — волны/окружности геометрически точные

**Что НЕ делать в python-pptx:**
- ❌ `RoundRectangle` с радиусом
- ❌ Round line caps (`a:ln cap="rnd"`)
- ❌ Тени, glow, blur эффекты
- ❌ Gradient fills (только сплошные)
- ❌ Italic на основном тексте

**Канонические правила приоритетов** (из эталонных слайдов):
1. **#222222** — основной для текста (титулы, body, иконки)
2. **#FFFFFF** — текст **только на тёмных слайдах/плашках** (графит/чёрный фон)
3. **#F2F2F2** — тонкие разделители, неактивные элементы
4. **#26D07C** — **акцент через цветной ЭЛЕМЕНТ** (плашка, divider, подчёркивание, footer-плашка, заливка chart-серии), **не цвет текста**
5. **Yellow / Purple / Blue** — для цветовой кодировки процессов (как заливка элементов, не текст)

⚠️ **Зелёный — НЕ цвет body!** Только акцент (≤30% площади слайда).

### 🔴 Правило контраста текста (canonical 2026-05-29, Problem #2)

**Текст ВСЕГДА #222222 на белом фоне И на зелёном фоне.** Запрещено:
- ❌ зелёный текст на белом (`#26D07C` как цвет букв/цифр) — плохо читается;
- ❌ белый текст на зелёной плашке (`white-on-green`) — плохо читается.

Это распространяется и на **крупные акцентные цифры**: главная KPI-цифра (199pt)
и divider-цифра тоже **#222222**, не зелёные (user-decision 2026-05-29).

**Как делать акцент** (вместо зелёного текста): выносить акцент в отдельный
**цветной элемент** — зелёная плашка-подчёркивание под/над цифрой, зелёный
divider, зелёный footer-плашка, зелёная заливка блока (с графитовым текстом).

Исключение для белого текста: **тёмные слайды/плашки** (фон графит `#222222` /
чёрный) — там текст белый, это нормально (высокий контраст). Зелёный текст не
допускается нигде.

## 6. Typography

### Шрифт-семейство (canonical — user-confirmed)

**Главный шрифт = SB Sans Display** (одно семейство, два начертания: Regular + SemiBold).

⚠️ Брендбук на стр.31 показывает 3 подсемейства (Display/Text/Interface) — это **визуальная демонстрация вариаций**, не технические разные шрифты. На практике используем **SB Sans Display** для всего: заголовки, body, цифры. Различие только в начертании (Regular vs SemiBold) и размере.

| Шрифт (точное имя face) | Назначение |
|---|---|
| **SB Sans Display** | Body, описания, обычный текст (Regular) |
| **SB Sans Display Semibold** | Заголовки, акценты, KPI-цифры (эмфаза) |
| Verdana | Fallback (когда SB Sans недоступен — предустановлен в системе) |
| Pingfang (平方) | Только для иероглифов |
| SimSun | Fallback для иероглифов (никогда для не-иероглифного текста) |

🔴 **Эмфаза = SemiBold-face, НЕ bold-флаг (Problem #3, 2026-05-29):**
Жирность задаётся **именем начертания** `SB Sans Display Semibold` (этот face
встроен в шаблон, рендерится в PowerPoint на любой машине). **Никогда** не
ставить bold-флаг (`b=1`) на `SB Sans Display` — это даёт «фейковый» тяжёлый
жирный. Везде, где раньше был Bold, теперь начертание Semibold + bold=false.

### Точные параметры (из брендбука стр.32-34)

| Тип | Интерлиньяж | Трекинг |
|---|---|---|
| Заголовок (SB Sans Display) | **100%** | **0** |
| Подзаголовок (SB Sans Text) | **120%** | **0** |
| Body — обычный | 120% | 0 |
| Body — плотный | **100%** | **−2** |
| Большие цифры | **100%** | **−4 … −7** (отрицательный!) |

### Размеры из эталонных слайдов

| Слайд | Заголовок | Подзаг | Body | Цифры |
|---|---|---|---|---|
| Title (5) | 88pt | 28pt | — | — |
| Title (8) тёмный | 44pt | 20pt | — | — |
| Divider Green (12) | 54pt | — | — | 66pt |
| Divider Dark (13) | 96pt White | — | — | 44pt Green |
| Content | 20-32pt | 20pt | 14pt | — |
| KPI big | — | — | 12pt | **199pt** + 88pt |
| Callout (42) | — | — | — | 44pt |

### Правила вёрстки текста
- **Расстояние между блоками** = высота заглавной буквы заголовка
- Заголовок 2-3 строки **без переносов**
- **Жирный = только через начертание `SB Sans Display Semibold`**, не bold-флаг (Problem #3)
- **Размер контента: стандарт 16pt, комфортный минимум 12pt** (Problem #5). Меньше 12pt — только при сильной перегрузке (сначала Overflow Strategy). < 10pt — недопустимо.

## 9. Portal — БАЗОВЫЙ бренд-элемент

Из брендбука стр.35-37:

> «Портал — базовый бренд-элемент, создающий образ современной, технологичной компании и позволяющий собирать все виды дизайн-элементов в упорядоченную, гармоничную структуру.»

**Конструкция:**
1. Основа — прямоугольник
2. Усложнение — повторение со сдвигом → **ступенчатая** форма
3. Несколько прямоугольников — сложная динамичная композиция

**Трансформации:** поворот 90/180°, отражение по вертикали/горизонтали.

**Заполнение:** любой цвет палитры **или** паттерн.

**Использование:**
- Подложка для логоблока с дескриптором
- Контейнер для изображений (фото / 3D)
- Самостоятельный декоративный элемент
- Подложка для текстового блока

**В шаблоне:** портал визуально присутствует в slide 12 (зелёный с бракет-рамкой), 13 (тёмный аналог), 95 (зелёный лого).

## 10. Linear illustrations

### Базовые символы (стр. 41-43)

| Символ | Значение | Применение |
|---|---|---|
| **Cloud (облако)** | Cloud.ru / cloud-инфраструктура | Фоновый декор, callout |
| **Star (звезда ✦)** | AI / magic / искусственный интеллект | Акценты, метки AI |

### Технические правила
- **Толщина линии 2/4/6 px** (кратна микромодулю!) — 1px только в исключение
- **До 2 цветов** на одну иллюстрацию
- Цвета — из брендовых палитр
- Стрелка может быть с **подложкой-квадратом 45°** другого цвета

### Тематические наборы
Лабораторные приборы, машины, мониторы — для спецпроектов. Сохранять стилистику основного набора.

### Псевдо-3D линейные (стр. 47)
Сложная техника: только в составе **группового изображения** с паттерном и порталом. Для важных статей и маркетинга.

## 12. Composition principles

Из брендбука стр.28:

> «При создании подобных решений большое значение имеют композиционные приёмы (акцент, контраст, ритм и т.д.).»

### 3 главных правила композиции

1. **Акцент.** Один главный элемент. Если 2 — ошибка.
2. **Контраст.** Светлое/тёмное, паттерн/сплошной, мелкий/крупный текст.
3. **Ритм.** Чередование типов слайдов: title → divider → content → callout → KPI.

### Quadrato-modular layout
- Сетка кратна микромодулю 2 px
- Сторона квадрата = кратно 10 px
- Гармоничное распределение масс

## 14. Применение в build-pipeline

### Цветовая логика выбора donor'а
- **Tone светлый (default):** title 5/6 → divider 12 (Green) → content 28/29/34 → callout 42 → KPI 43 → finale 95
- **Tone тёмный (premium):** title 7/8 → divider 13 (Dark) → content 22/57 → callout 41 → KPI 44 → finale 93
- **Mixed:** чередовать divider Green ↔ Dark для ритма

### Распределение акцент-цветов в KPI/инфографике
- **Только 1 цифра/блок зелёный** — главный показатель
- Остальные — чёрный текст
- Если KPI отрицательный (риск/потери) — НЕ красным, а серым приглушённым

### Что НЕ делать
- ❌ Bullet points (используй структурные блоки/колонки шаблона)
- ❌ Эмодзи (📊🚀 — заменять на иконки/линейные illust или удалять)
- ❌ Italic, Underline
- ❌ Mixed font weight в одном run
- ❌ Не-палитровые цвета
- ❌ Round corners, drop shadows, glow

### Технические инсайты по шаблону (XML-инспекция)
1. **Декор callout 42 живёт в `slideLayout24.xml`** → достаточно `add_slide(layout=23)`, всё наследуется
2. **Декор divider 12 живёт на самом slide12.xml** (100 prstGeom!) → нужен **clone**
3. **KPI 43 имеет 18 prstGeom** на slide → **clone**
4. **Title 5 — 8 prstGeom + 3D PNG** → **clone** (картинка тащится автоматически)
5. **Logo finale 25/95** — декор в layout, можно `add_slide(layout=95)`

### Правило выбора между clone и add_slide(layout)
| Случай | Метод |
|---|---|
| Декор минимальный, в layout (callout, simple content) | `add_slide(layout)` — быстрее |
| Декор сложный на slide (divider с паттерном, KPI, title с 3D) | `clone_slide(donor_num)` — копируем shapes |

## 14b. Overflow strategy — что делать когда текст не помещается

User canonical (2026-05-02): 4 разрешённых стратегии в порядке предпочтения.

| # | Стратегия | Когда применять |
|---|---|---|
| 1 | **Другой донор** (предпочтительная) | Если в той же `tone_group` есть донор с большим slot capacity. Например title_white_with_3d (donor 5, max 30) → title_white_compact (donor 4, max 80) |
| 2 | **Разбить на несколько слайдов** | Если контент логически делится: длинный body → 2 слайда «1/2», «2/2». KPI с 4+ цифрами → 2 KPI-слайда. 6+ блоков → 2×3 |
| 3 | **Уменьшить кегль (20-30%)** | Минимальный override. 88pt → 70-72 (18-20%), 44pt → 32-36 (18-27%). НИКОГДА >30%. НИКОГДА <10pt |
| 4 | **Копи-эдитор** | Сократить текст без потери смысла, перевести в bullet-форму без знаков, разбить на title+subtitle |

### Что ЗАПРЕЩЕНО (главные ошибки)
- ❌ Override size_pt 44 → 199 (любой override >30%)
- ❌ Резать текст случайно без логики
- ❌ Менять шрифт на узкий fallback
- ❌ Переписывать donor XML-shapes по позициям

См. полные правила в `memory/feedback_overflow_strategy.md`.

## 15. Quick checklist для Verifier-агента

Перед финализацией каждого слайда проверить:

- [ ] Цвет каждого text-run в палитре §4
- [ ] Шрифт SB Sans (Display/Text/Interface) или Verdana fallback
- [ ] Размер ≥ 10pt
- [ ] **Один** главный элемент (акцент)
- [ ] Контраст с соседним слайдом по тону
- [ ] Заголовок ≤ 3 строк
- [ ] Микромодуль соблюдён (отступы кратны 10)
- [ ] Зелёный использован только как **акцент** (≤30% площади)
- [ ] Эмодзи отсутствуют
- [ ] Round corners / shadows / italic / underline отсутствуют
- [ ] Лого/footer на месте
- [ ] **Не сочетаются базовая и расширенная палитры**
