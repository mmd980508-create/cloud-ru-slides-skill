# Правила вёрстки ХАРД-СХЕМ (Cloud.ru) — canonical чеклист v2.5.1

Свод правил по итогам сессии 2026-06-05 (хард-схема Enterprise Router + Cloud
Firewall: 3 зоны, 6 вложенных таблиц, потоки, легенда). Применять ко всем
нагруженным схемам (`flow_diagram_native` с зонами/таблицами/потоками).
Эталон-реализация: `tests/hard_schema_build.py`.

Порядок работы: **полнота (ничего не терять) → сетка/оси → слои фреймов → анти-наезд
→ цвет → визуальная сверка**.

---

## 0. ПОЛНОТА КОНТЕНТА (сначала!) — agent 11
Перед вёрсткой инвентаризировать источник по 6 категориям и держать parity:
F рамки · N узлы/иконки · T таблицы(+все ячейки) · W подписи/слова · P пути/стрелки ·
L легенда. Правила: **каждая запись легенды имеет referent в теле**; сущность из
ячейки (ECS 3, Enterprise router) есть как узел; у стрелки оба конца реальны;
внутризонные потоки (ECS→Firewall) не терять; маркеры (✕) не выкидывать.
Девиз: **не упрощать — перенести ВСЁ, расставить, перекрасить.** В конце —
`_parity_check()` (счётчики source/output + legend⇒referent).

## 1. СЕТКА И ОСИ (воображаемые, в макете их нет)
- Строить ПО СЕТКЕ, а не координатами «на глаз».
- **Multi-zone → `auto_align=false`** (глобальный кластер стягивает таблицы разных
  зон и отрывает от подписей). `grid_snap=true` оставить. `align_rects` — только
  позиции (left/top), без ресайза.
- **Горизонтальные оси:** заголовки зон верхнего ряда — на ОДНОЙ линии (`HEAD_Y`);
  таблицы ряда — выровнены по ВЕРХУ (плашки-заголовки на одной линии, `T1`/`T2`);
  стрелки трафика — на фикс. горизонталях ряда (`ARROW1`/`ARROW2`).
- **Вертикальные оси:** края зон/таблиц + центры зазоров между зонами. Подписи
  attachment — отцентрованы в зазорах. Стрелки идут СТРОГО от края до края
  (зона↔таблица), без перелёта внутрь панели.
- **Иконки-узлы** центрируются по своей таблице (или выровнены по её верху) — единый
  размер.
- Высоту таблиц считать ЗЕРКАЛОМ формулы рендерера (см. `table_h` в эталоне), иначе
  центрирование/вписывание мимо.

## 2. ТАБЛИЦЫ-В-СХЕМЕ (единый компонент)
- **Единая типографика** всех однотипных таблиц: title 10 / header 9 / data 9 pt,
  одни мин-высоты → плашки не «гуляют», заголовки на одном месте внутри плашек.
- Текст ячеек — по ЛЕВОМУ + ВЕРХНЕМУ краю. Поля 10/8/4/3 (не липнет к рамке).
- Высота плашки-заголовка считается по ПОЛНОЙ ширине таблицы (не по 1-й колонке —
  иначе ошибочно ×2).
- **Ширину колонок подбирать под самый длинный текст** (без переноса), иначе строка
  в 2 строки → таблица выше → вылезает за subnet/SG/панель.
- Таблица рядом с другими объектами → С границами; одиночная таблица-слайд → zebra.

## 3. СЛОИ ФРЕЙМОВ (иерархия границ областей)
Границы выделять ПРЕИМУЩЕСТВЕННО ЗАЛИВКОЙ. Приоритет: **(1) заливка → (2) сплошная
0.75pt `#D3D3D3` → (3) пунктир**. Заливки ЧЕРЕДУЮТСЯ от белого слайда:
- **Зона (VPC)** = `panel` Gray 100 `#F2F2F2` + нижний штрих `#D3D3D3` 1pt (только низ).
- **Вложенная группа (security group)** = `panel` White.
- **Глубже (subnet)** = пунктир `#D3D3D3` (тире=пробел=2pt, 0.75pt).
- **Узел-компонент (ECS/Firewall)** = white/gray + ЗЕЛЁНЫЙ нижний штрих `#26D07C` (акцент).
- **Максимум ОДИН пунктирный слой** (несколько боксов на одном уровне — ок).
- НЕ инвертировать порядок; НЕ добавлять серый фон-стейдж (даёт серые «колонки»).
- Нижние штрихи — на СЕРЫХ фреймах/узлах; белые группы — без штриха.
- **ОДНА ЛОГИКА:** серую рамку получают ТОЛЬКО однотипные зоны (VPC). Разнородная
  сущность (Enterprise router — не VPC) серой рамки НЕ получает: только заголовок +
  её таблицы. Рамка = маркер одного класса зон.

## 4. ЦВЕТ
- Серый-доминант + зелёные акценты. Другой цвет — ТОЛЬКО для смысловой
  дифференциации по легенде (зелёная associate-линия, красный ✕ blocked).
- Штрихи фреймов = `#D3D3D3` (graphite-50). Стрелки-трафик = graphite `#434343`
  (это контент, не граница).

## 5. ПУТИ / СТРЕЛКИ
- Только горизонтали/вертикали (диагонали запрещены → ортогональный Z-маршрут).
- Открытая галочка size 8 (`w=lg,len=med`), 1pt. Цвет из конфига для смысловых путей.
- Request = solid graphite; Response = dashed; Associate = green (без головы);
  Blocked = красный ✕ на сегменте. Внутризонный поток (ECS↔Firewall) — вертикальный.

## 6. АНТИ-НАЕЗД (детектор после вёрстки — обязателен)
Прогнать проверку пересечений значимых элементов. Типовые наезды:
- иконка-узел ↔ таблица: таблица начинается ПОСЛЕ иконки + зазор 8px
  (`x = subnet_left + inset + ICON_W + 8`);
- таблица ↔ её рамка: колонки под контент → таблица вмещается в subnet/SG/панель;
- подпись легенды ↔ образец: подписи узкие (~170px), бокс не перекрывает линию/✕;
- подпись зоны ↔ контент: заголовок не накрывает таблицу.
