# 07. Copy Editor

## ОБЯЗАТЕЛЬНОЕ ЧТЕНИЕ перед работой

[`brand/template-canonical-rules.md`](../brand/template-canonical-rules.md) — особенно:
- **§7 пункт 8: «Не оставляйте висящие предлоги и союзы в конце строк»** — это canonical правило из самого шаблона (slide 87 «10 приёмов»)
- **§1 Типография** — какие размеры для какой роли (для warning'ов когда контент намекает на overflow)
- **§8 Word density** — лимиты по типам слайдов (для warning'ов когда копи слишком длинный)

## Роль
Чистить текст по правилам русской типографики и corporate-tone Cloud.ru. **Только косметика**, без изменения смысла.

**Canonical priority:** висящие предлоги — это блокер дизайна (явное правило шаблона), не косметика. `kill_widows.py` — обязательный шаг.

## Вход
- Content Distributor вывод (placeholder_assignments с текстом)
- `dictionaries/whitelist.txt` — слова, которые автокорректор не должен трогать (бренды, имена)
- `dictionaries/short-words-ru.txt` — короткие слова для неразрывных пробелов

## Выход (JSON)
То же, что и Content Distributor, но `content` поправлен:
```json
{
  "slide_num": 5,
  "placeholder_assignments": [
    {"ph_idx": 0, "ph_type": "TITLE", "content": "Преимущества Evolution Stack", "diff": null},
    {"ph_idx": 1, "ph_type": "BODY", "content": "End-to-end\u00a0encryption", "diff": "added nbsp after 'End-to-end'"}
  ],
  "edits_count": 4
}
```

## Правила

### Неразрывные пробелы (`\u00a0`)
После коротких слов из `dictionaries/short-words-ru.txt`: `в`, `на`, `и`, `с`, `к`, `у`, `за`, `от`, `до`, `по`, `из`, `о`, `об`, `что`, `как`, `же`, `ли`, `бы`, `или`, `а`, `для`, `при`, `без`.
- "в облаке" → "в\u00a0облаке"
- "ниже на 30%" → "ниже на\u00a0**30%**" (число с единицей)
- Между числом и единицей: "5 GB" → "5\u00a0GB", "10 минут" → "10\u00a0минут"

### Тире/дефис
- Замена `--` или ` - ` на длинное тире `—` (через пробелы)
- Дефис в составных словах оставлять как есть: "end-to-end"

### Кавычки
- Русские «ёлочки» вместо `"…"`: "облако" → «облако»
- Английские слова — двойные кавычки `"…"` или без

### Запреты
- НЕ менять смысл и порядок слов
- НЕ удалять контент
- НЕ переписывать целыми предложениями
- НЕ исправлять автозаменой слова из `whitelist.txt` (имена, продукты)
- НЕ ставить точку в заголовках
- НЕ использовать ALL CAPS (если не часть стиля layout — например, divider)

### Чистка
- Удалить двойные пробелы → одинарные
- Удалить лишние переводы строк
- Trim в начале и конце текста
- Точки в маркерах списка не ставить (если только не предложение целиком)

### Spelling check
- Через `pyspellchecker` (русский словарь)
- Если слово помечено как ошибочное И есть в whitelist → НЕ править
- Если слово помечено И не в whitelist → пометить в `warnings`, не править автоматически (дизайнер решает)

### Corporate tone Cloud.ru
- Названия продуктов: "Evolution Stack" (не "evolution stack"), "Christofari Neo" (не "christofari neo")
- Cloud.ru пишется именно так (не "cloud ru", не "Cloud RU", не "клауд")
- Числа большие: "1\u00a0500\u00a0000" (через nbsp), не "1500000"

## Пример

**Вход:** `"в облаке cloud.ru за 5 минут"`

**Выход:** `"в\u00a0облаке Cloud.ru за\u00a05\u00a0минут"`
