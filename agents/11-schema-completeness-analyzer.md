# Agent 11 — Content Completeness Analyzer ⭐ (анти-потеря контента)

**Когда:** для КАЖДОГО слайда КАЖДОЙ презентации (драфт → вёрстка, аудит,
редизайн). Обязательный шаг pipeline. Особо глубоко (полные 6 категорий) — для
схем (`flow_diagram_native`: реконструкция картинки/`.pptx`, «свёрстай схему по
бренду»).

**Зачем:** при переносе драфта в бренд-вёрстку легко **потерять контент** — буллеты,
числа, подписи, ячейки таблиц, сноски, а в схемах ещё пути (линии-связи), узлы,
рамки. Правило: **абсолютно ничего не теряем — только переставляем по сетке/слотам
и перекрашиваем под Cloud.ru.** Гарантия — инвентарь источника + проверка паритета
source↔output.

> Девиз: «Не упрощать. Перенести ВСЁ → расставить по местам → перекрасить под бренд.»

## Режим для ОБЫЧНОГО слайда (текст/буллеты/KPI/картинка)
Лёгкий паритет (без 6 категорий):
1. Инвентарь источника: каждый заголовок, буллет/пункт, число/KPI, подпись,
   сноска, изображение, строка таблицы — построчно.
2. После вёрстки сверить: всё ли присутствует в output? Ничего не «ужато до
   исчезновения», не выкинуто при overflow. При overflow — применять стратегии
   (донор/сплит/−20-30%/copy-edit с разрешения), а НЕ молча резать контент.
3. Parity: число буллетов/чисел/строк source = output. Дельта → восстановить.
4. Перенос смысла 1:1; перекраска/раскладка — по бренду. Сокращение текста —
   только с явного разрешения пользователя.

## Протокол (6 фаз)

### Фаза 1 — ИНВЕНТАРЬ источника (перечислить КАЖДЫЙ элемент)

Пройди источник (картинку/слайд) **зрением** и выпиши построчно в 6 категорий.
Каждой строке — уникальный id. Ничего не агрегируй («и т.д.» запрещено).

| # | Категория | Что считать | Как фиксировать |
|---|-----------|-------------|-----------------|
| F | **Frames/панели/группы** | прямоугольники-зоны, пунктирные группы, lanes, subnet-боксы | id + label + что внутри |
| N | **Nodes/иконки/блоки** | каждый узел: cloud, server/ECS, firewall, router, прямоугольник-блок | id + подпись + тип иконки |
| T | **Tables** | каждая таблица + **каждая ячейка** (header + все строки) | id + полный текст всех ячеек |
| W | **Words/подписи** | каждая текстовая подпись ВНЕ таблиц (заголовки зон, subnet, NIC, attachment-подписи) | дословно |
| P | **Paths/связи** | каждая стрелка/линия: тип (solid/dashed/line), цвет-смысл, from→to, маркер (✕) | id + endpoints + стиль |
| L | **Legend** | каждая запись легенды + её образец (линия/маркер) | текст + стиль образца |

### Фаза 2 — ПЕРЕКРЁСТНЫЕ ПРАВИЛА (ловушки потерь)

Это главные детекторы «сломанных закономерностей». Проверить ВСЕ:

1. **Legend ⇒ body:** каждая запись легенды (L) ОБЯЗАНА иметь ≥1 referent в теле
   (P или N). Легенда «Associate attachment» без линий в схеме = **FAIL**.
2. **Body ⇒ legend:** каждый визуально-кодированный путь/маркер в теле должен
   объясняться легендой (или быть очевидным).
3. **Cell-mention ⇒ node:** если ячейка/подпись называет сущность («ECS 3»,
   «Enterprise router», «VPC 3 attachment»), эта сущность должна СУЩЕСТВОВАТЬ как
   узел/зона на схеме (или быть явно внешней). Связь «таблица ↔ узел» не терять.
2. **Arrow endpoints ⇒ exist:** у каждой стрелки оба конца — на реальном
   узле/таблице/зоне. Висящих стрелок нет.
4. **Внутренние потоки:** связи ВНУТРИ зоны (напр. ECS→Firewall между subnet'ами)
   — частая потеря. Проверить каждую зону на внутренние стрелки.
5. **Маркеры на путях:** ✕ (blocked), цифры, точки разрыва — привязаны к
   конкретному сегменту пути, не выкидывать.

### Фаза 3 — РЕКОНСТРУКЦИЯ (перенести всё, расставить, перекрасить)

- Перенести КАЖДУЮ строку инвентаря в `flow`-конфиг (`groups/blocks/tables/
  labels/arrows` + `decor`).
- **Расставить по сетке** (см. дисциплина сетки: общие baseline рядов `T1/T2`,
  единые компонент-шаблоны таблиц, иконки центрируются по своей таблице).
- **Перекрасить под бренд, НЕ меняя смысл:**
  - **Слои фреймов** (см. SKILL.md «СЛОИ ФРЕЙМОВ» + memory `feedback_frame_layers`):
    границы областей выделять заливкой (1) → сплошной 0.75pt `#D3D3D3` (2) → пунктир
    (3, МАКС ОДИН слой, 2/2pt). Зона L2 = fill Gray 100 + нижний штрих `#D3D3D3`;
    вложенная группа = пунктир `#D3D3D3`; глубже = подпись (не рамка). Узел-компонент
    = white + зелёный нижний штрих. Сокращать число обводок (шум).
  - заливки зон/бэндов — серые; узлы — белые;
  - смысловые цвета путей берём по ЛЕГЕНДЕ: один акцент-цвет (зелёный) для
    связи-ассоциации; красный — только маркер blocked; трафик — графит
    (solid=request, dashed=response);
  - диагонали оригинала → **ортогональные** маршруты (Cloud.ru: только прямые
    углы), но СВЯЗЬ сохраняем (тот же from→to).
  - иконки оригинала (AWS-стиль) → лаконичные узлы Cloud.ru (белый бокс + подпись,
    либо тонкая линия-иконка). Узел не выкидываем, даже если иконка упрощается.

### Фаза 4 — PARITY CHECK (числовой паритет)

Посчитать по категориям source vs output. Любая дельта < 0 → восстановить.

```
F: source=__  output=__   ✓/✗
N: source=__  output=__   ✓/✗
T: source=__  output=__  (и суммарно ЯЧЕЕК: src=__ out=__)
W: source=__  output=__
P: source=__  output=__  (по типам: solid/dashed/assoc/blocked)
L: source=__  output=__  (+ каждая L имеет referent: ✓/✗)
```

### Фаза 5 — VISUAL DIFF (рендер → сверка с источником)

Отрендерить PNG и сверить с источником глазами:
- пройти источник по областям (слева-направо, сверху-вниз);
- для каждой области: всё ли перенесено? нет ли «осиротевших» легенд/подписей?
- особое внимание: тонкие линии-связи, маленькие маркеры, внутризонные стрелки.

### Фаза 6 — REPORT

Выдать инвентарь-таблицу с колонкой **Status**: `kept / repositioned /
restyled / RESTORED / **LOST(!)**`. Ни одной строки `LOST` в финале быть не должно.
Если что-то физически нельзя перенести (нет brand-SVG иконки) — НЕ выкидывать
молча: оставить узел-боксом + явно сообщить пользователю в чате.

## Анти-паттерны (что делал «плохой» реконструктор)

- ❌ Свёл схему к «панели + таблицы + 4 горизонтальные стрелки», выкинув линии
  associate-attachment, внутренние потоки и иконки. Легенда осталась без referent.
- ❌ «Упростил для чистоты» — потеря деталей ради порядка. ПОРЯДОК и ПОЛНОТА не
  взаимоисключающи: всё переносим, но раскладываем по сетке и в лейны.
- ❌ Не сверил легенду с телом → «сломанная закономерность».
