# Flow diagrams — эталонные примеры

Референсные PNG-рендеры для `slide_type: flow_diagram_native` (v1.6+).

## Файлы

| Файл | Описание |
|---|---|
| `sgr_schema.png` | SGR (Schema-Guided Reasoning) — 4-колоночная схема phase blocks + LLM calls + arrows + 2 punктирные группы Phase 1 / Phase 2 + schema-example блок |
| `analytics_pipeline.png` | Pipeline аналитики — 4 main блока линейкой, развилка стрелок к 3 outputs справа (PM / ML / Аналитики) |

## Что считается «PASS» по canonical-стилю

- Блоки **серые `#F2F2F2`** (не цветные), текст **графит `#222222`**
- Стрелки **чёрные 1pt** с маленьким треугольным наконечником (`w=sm len=sm`)
- Пунктирные рамки `#888888` для группировки фаз
- Заголовок слайда **20pt SemiBold CAPS top-left** (одинаковый стиль с content слайдами шаблона)
- Тонкая `#CCCCCC` горизонтальная линия под заголовком на y=110
- Декор: зелёные L-уголки с диагональю — в одном из нижних углов (свободный)
- **Без drop-shadow и glow** на блоках (canonical §3 brand-rules)
- Минимальный размер текста 10pt

## Как читать координаты

Slide canvas — `1280 × 720 px`. Все координаты в `flow_config.blocks[]` —
в пикселях. `flow_renderer.px(v)` конвертит в EMU.

## Воспроизвести

Исходный экспериментальный скрипт (с конкретными координатами обоих
референсных слайдов) — `output/flow_diagrams/build_flow.py`. Не входит в репо
(в .gitignore), но если нужен — лежит в локальной рабочей копии.
