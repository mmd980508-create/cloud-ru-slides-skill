# Cloud.ru Template 2026 — Семантический каталог 102 layouts

Шаблон: `Cloud.ru_Template_2026.pptx` (29MB, 88 слайдов превью, 102 master layouts).
Размер слайда: 1280×720 px @ 96dpi (EMU 12192000×6858000), 16:9.
Полный дамп (с координатами placeholder'ов в EMU/px) — `template-layouts-dump.json`.

> **Layout Designer:** при выборе layout всегда ссылайся на `idx`, не на имя — имена содержат пробелы и дубли (`1_Команда 10` vs `Команда 10`). Имена с префиксом `1_` — копии для другого master/section.

## Decision tree (когда какой layout)

```
Что нужно?
├─ Открывающий слайд → title (idx 0-7, 52, 53, 98, 99)
├─ Раздел / глава    → divider (idx 8, 9, 54-56)
├─ Чистый текст      → text (idx 24, 27, 33, 34, 44, 46, 47, 66, 75, 87)
├─ Несколько колонок → multicolumn (idx 25, 28-32, 68-74)
├─ Картинки/фото     → image (20, 21, 22, 37, 40-45, 78-81, 86, 88-92, 94)
├─ Команда           → team (48-51, 82-85)
├─ Процесс / шаги    → timeline (38, 39, 76, 77)
├─ Большая таблица   → table (35)
├─ Важный блок       → callout (23, 67)
├─ Декор. фон        → pattern_bg (12-19, 58-65)
└─ Логотип крупно    → logo (93, 95)
```

## Совет Layout Designer'у

**Антипаттерн:** не выбирай layouts по имени методом подстроки — есть "1_"-копии. Всегда работай через **idx**.

**Запрет ритма:** не использовать одинаковую категорию >2 раз подряд. Чередуй: text → multicolumn → image → divider → text…

**Тёмные/светлые:** в одной презентации допустимы оба, но переход — через divider. Не миксовать на соседних слайдах без разделителя.
